# Pair Programming Report

## 1. Introduction
This project adopts Pair Programming to improve code quality, reduce bugs, and enhance collaboration efficiency. Two team members worked together using the driver–navigator model throughout the development process.

---

## 2. Roles & Responsibilities

**Driver**
- Responsible for writing code and implementing features  
- Focuses on syntax and immediate implementation  

**Navigator**
- Reviews code in real time  
- Suggests improvements and detects potential issues  
- Ensures overall logic and structure are correct  

**Role Switching**
- Roles were switched every 30–60 minutes to ensure equal participation and knowledge sharing  

---

## 3. Workflow

1. Requirement discussion and clarification  
2. Task breakdown into smaller components  
3. Real-time pair coding (Driver + Navigator)  
4. Continuous review and feedback  
5. Commit and push changes using Git  

---

## 4. Tools Used

- Visual Studio Code (code editor)  
- Git (version control)  
- GitHub (repository management)  
- Online communication tool (Zoom)  

---

## 5. Benefits

- Improved code quality through continuous review  
- Reduced number of bugs and errors  
- Faster debugging process  
- Better knowledge sharing between team members  
- Increased collaboration and communication  

---

## 6. Challenges

- Differences in coding styles between team members  
- Scheduling conflicts for pair sessions  
- Communication gaps during complex implementations  

---

## 7. Improvements

- Improve time management for pair sessions  
- Define clearer task boundaries before coding  
- Use more structured communication during development  

---

## 8. Conclusion

Pair Programming proved to be an effective development approach in this project. It enhanced collaboration, improved code reliability, and helped both members gain a deeper understanding of the system.
